Evaluate whether the candidate image is ready to use.

Compare the candidate against the original source image and the intended
teaching goal.

Return JSON only with these fields:
- verdict: "pass" or "fail"
- summary: one short sentence
- issues: array of objects with severity, observation, and repair_instruction
- next_repair_issue: one concrete sentence to pass to repair-image, or an empty string if verdict is pass
- keep_parts: array of visual details that should be preserved on the next repair turn

Judge these failure modes:
- source facts changed
- important regions cropped or distorted
- labels cover the thing they explain
- arrows or highlights target the wrong area
- fake text, fake logos, fake card names, fake UI, or unrelated objects appear
- visual style clashes with the destination document
- the image looks acceptable alone but would fail at document size

Prefer a repair instruction that changes one local issue only.
